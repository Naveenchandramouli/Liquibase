Step 1: Update linux machine

  • dnf -y update

Shell

[root@liquibase ~]# dnf -y update


Step 2: Install Java

  • Liquibase is a Java-based tool, so we need to install Java on your CentOS system.
  • You can check if Java is already installed by running in linux shell by typing.

Shell

[root@liquibase ~]# java -version


  • If Java is not installed, you can install OpenJDK, a popular open-source Java implementation, using the following commands.

Shell

[root@liquibase ~]# dnf install java-11-openjdk-devel


Step 3: Download Liquibase

  • Visit the Liquibase download page at https://www.liquibase.org/download.
  • Select the latest version of Liquibase and download in ZIP or TAR.GZ format for your platform.

Step 4: Extract Liquibase

  • Certainly, you can use either the `unzip` command for ZIP files or the `tar` command for TAR.GZ files to extract the contents of the Liquibase binary distribution, depending on the format you downloaded.
  • Here are the commands for both formats:

    • For ZIP Format:

Shell

[root@liquibase ~]# unzip liquibase-version-bin.zip

  • Replace version with the actual version number you downloaded.

    • For TAR.GZ Format:

Shell

[root@liquibase ~]# tar -xzvf liquibase-version-bin.tar.gz

  • Replace version with the actual version number you downloaded.
  • Choose the appropriate command based on the format of the Liquibase binary distribution you downloaded. Once extracted, you can proceed with configuration of Liquibase.

Step 5: Set Up Environment Variables

  • You can add Liquibase to your system path and set the LIQUIBASE_HOME environment variable.
  • Add the following lines to your ~/.bashrc or ~/.bash_profile file or equivalent if you use a different shell.

Shell

[root@liquibase ~]# vi ~/.bashrc

export LIQUIBASE_HOME=/path/to/liquibase
export PATH=$PATH:$LIQUIBASE_HOME

  • Replace /path/to/liquibase with the actual path to the Liquibase directory.

Step 6: Verify Installation

  • Reload your shell profile or open a new terminal window to apply the environment variable changes.
  • You can now verify the Liquibase installation by running.

Shell

[root@liquibase ~]# liquibase --version


Step 7: Create a Liquibase Properties File

  • Choose a directory where you want to set up your Liquibase project. You can create a new directory for this purpose.
  • Create a `liquibase.properties` file in your project directory or any other location where you plan to manage your database changes.
  • Here's a sample `liquibase.properties` file.

Shell

[root@liquibase ~]# vi liquibase.properties
Liquibase.properties:
# Database configuration
driver: com.mysql.cj.jdbc.Driver
url: jdbc:mysql://localhost:3306/mydatabase
username: dbuser
password: dbpassword

  • Replace the database configuration details with your database's connection information. Modify the `changeLogFile` to point to your changelog XML or YAML file.

Step 8: Create a Changelog File

  • Create a changelog XML or YAML file that describes the changes you want to make to your database. Here's an example of a simple changelog.yaml file.

Shell

[root@liquibase ~]# vi changelog.yaml

databaseChangeLog:
  - changeSet:
     id: create-bank-table
     author: bankingapp
     changes:
       - sqlFile:
          path: changesets/create-bank-table.sql
  - changeSet:
     id: create-account-table
     author: bankingapp
     changes:
       - sqlFile:
          path: changesets/create-account-table.sql


Step 9: External changesets

  • Certainly, if you want to create external SQL changesets for use with Liquibase in SQL format, you can create individual SQL files for each changeset. Each SQL file will contain the SQL statements you want to execute.
  • Here's an example of two external SQL changesets in SQL format.

Shell

[root@liquibase ~]# vi create-bank-table.sql

CREATE TABLE Bank (
name VARCHAR2(100) NOT NULL,
location VARCHAR2(255),
created_date DATE DEFAULT SYSDATE
);

Shell

[root@liquibase ~]# vi create-account-table.sql

CREATE TABLE Account (
account_number VARCHAR2(20) NOT NULL UNIQUE,
balance NUMBER(10, 2) DEFAULT 0,
owner_name VARCHAR2(100) NOT NULL,
created_date DATE DEFAULT SYSDATE
);

  • Each SQL file represents a single changeset.
  • The comments provide metadata such as the author and description.
  • The SQL statements inside the files define the database changes.
  • You can create as many SQL changeset files as needed for your Liquibase project.
  • These files can contain various SQL statements like table creations, index additions, data modifications, etc., depending on your database schema evolution requirements.
  • After creating these external SQL changesets, you can reference them in your Liquibase YAML or XML changelog file.
  • The sqlFile changeset within the YAML/XML changelog should specify the path to the external SQL file, and Liquibase will execute the SQL statements in those files when you run the liquibase update command.

Step 10: Running Liquibase Commands

  • You can now use Liquibase commands to manage your database schema. For example, to apply changes from your changelog file, use the following command.

Shell

[root@liquibase ~]# liquibase update

  • Remember to replace the placeholder values in the `liquibase.properties` file and the changelog file with your actual database and changelog details.
  • This guide covers the basic installation and configuration of Liquibase on CentOS. Depending on your specific database and use case, you may need to install database-specific drivers and configure Liquibase accordingly.
